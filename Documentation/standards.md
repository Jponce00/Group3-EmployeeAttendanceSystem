## Coding Standards

- Use clear lowercase function names with underscores, such as add_record().
- Use clear variable names, such as employee_id, employee_name, and attendance_status.
- Use 4 spaces for indentation.
- Add comments when the code may be difficult to understand.
- Each function must include a docstring explaining its purpose, parameters, return value, author, and status.
- Attendance status values must be written as Present, Absent, or On Leave.
- The program should manage up to 25 employee attendance records.

## Git Workflow

- The team will use the main branch for final approved files.
- Each team member will make at least one commit and push.
- Commit messages should be short and clear.
- Team members should review files before the final report is submitted.
- Major changes should be discussed before being added.

## Team Procedures

- Team members will communicate through text message, email, or class communication.
- Each member should respond within 24 hours when possible.
- Tasks will be divided so each member contributes to the project.
- If there is a conflict or confusion, the team will discuss it respectfully and choose the option that best follows the assignment instructions.
- The team will check GitHub before submission to make sure all required files are included.
